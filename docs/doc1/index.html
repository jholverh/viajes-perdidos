Viajes no realizados

Este aplicativo o herramienta web el cual se encuentra dentro del sitio web VISION BRT, fue realizado en el framework Django. El Backend fue realizado en Python, mientras que el Frontend está programado en Javascript utilizando HTML5 y CSS.

Diagrama básico de funcionamiento

Estructura y Directorio del aplicativo

Cabe resaltar que la mayoría de esta estructura y contenido de los archivos vienen dados por defecto, dado que son plantillas estándares. Se recomienda visitar este sitio.

Para mayor información refiérase al directorio y archivos en mención, los cuales cuentan con los comentarios respectivos. Se recomienda no modificar líneas de Código si no se tiene completa claridad sobre su utilidad en la aplicación web.

Backend y FrontEnd

Una vez se instala el framework Django y se genere la plantilla tipo para aplicación web se genera la estructura mostrada

mysite

En este subdirectorio destacan los archivos “settings.py” y “urls.py”. Settings.py: En este archivo se encuentran las instrucciones de configuración del backend desde el nivel general.

Urls.py: En el se encuentran las rutas hacia el backend y al login del sitio.

Servicios: BigQuery y GCloud (BD visión)

Estas configuraciones se encuentran dentro del archivo “settings.py”. El archivo con extensión “.json” se debe descargar desde Google Cloud Platform y almacenarse en el mismo directorio de la aplicación. En Bigquery se encuentra almacenada la información de cada viaje desde aproximadamente 2017, y es desde donde se obtiene la información histórica, mientras que en postgresql cloud se encuentra la información de los viajes del día en una tabla volátil.

Backend: /repvperd

Logic.py: Este archivo es el que contiene toda la lógica del backend. En él se encuentran las funciones que organizan los datos y los combinan para generar la información que será mostrada en el frontend. Por lo general llama las funciones contenidas en gcloud cuyos detalles se mostrarán posteriormente.

Views.py: Aquí se encuentran las vistas en las cuales se recogen las funciones del archivo logic y se disponen en las urls definidas para consulta final. Recibe los parámetros de entrada para la consulta, así como los requerimientos de la aplicación (login y caché).

Urls.py: Son las direcciones desde las cuales el backend dispone la información para consulta desde el frontend.

Frontend

/repvperd/static

Contiene los archivos y librerías necesarias para que se visualicen correctamente en el frontend. Estas se encuentran al inicio de los archivos HTML (en el head y el inicio del body)

Css: Hojas de estilo en cascada necesarias para los componentes del frontend.

Js: Librerías de javascript, plugins o complementos que contienen la programación de los componentes de las páginas (multiselector, script con instrucciones de visualización, renderizadores, jquery y pivot table).

/templates/repvperd: Aquí se encuentran los dos archivos HTML que contienen las pantallas que visualiza el usuario: El reporte diario y el reporte histórico de los viajes no realizados.

Cuando se hace clic en el botón consultar, se activan las funciones de envío de parámetros de cada “filtro” o lista de selección múltiple dentro de la función reload()

onclick="reload(enviarTipoServ(), enviarFechasSel(), enviarConcesSel(), enviarMotivSel(), enviarRutaSel())" >Consultar</button>

function reload(tipo_servi, fechass, concesionarioss, motivoss, rutass){
  var horamin = '03:00:00'
  var horaact = '23:59:00'
  var url_data ='http://' +window.location.host+'/repvperd/vjhistoria/datosjson'
  var fechass = fechass.toString()
  var concesionarioss = concesionarioss.toString()
  var motivoss = motivoss.toString()
  var rutass = rutass.toString()
  var tipo_servi = tipo_servi.toString()
  $.ajax({
      type: "POST",
      url: url_data,
      data: {
        csrfmiddlewaretoken: '{{csrf_token}}',
        fec: fechass,
        con: concesionarioss,
        ts: tipo_servi,
        mot: motivoss,
        ghi: horamin,
        ghf: horaact,
        rut: rutass
      },
      success: function(response){
        response = JSON.parse(response)
        var derivadores = $.pivotUtilities.derivers;
        $("#output").pivotUI(response, {
            cols: ["tiposervicio"], 
            rows: ["fecha","concesionario"],
            vals: ["v_perd"],
            aggregatorName: "Sum",
            rendererName: "Table",
            skipEmptyLines: false,
            renderers: $.extend($.pivotUtilities.renderers, 
                                $.pivotUtilities.plotly_renderers
                                )
                });
            $('#btn-one').html('<span class="spinner-border spinner-border-sm mr-2" role="status" style="display:none"></span>');
            $('#btn-one').html('Consultar')
            closeNav()
            },
      error: function (jqXHR, exception){
        alert('Ingrese todos los parámetros requeridos')
        $('#btn-one').html('<span class="spinner-border spinner-border-sm mr-2" role="status" style="display:none"></span>');
        $('#btn-one').html('Consultar')
        }     
      });
      
    }

El envío de los datos se realiza a través del método POST, y se genera un error cuando el usuario no ha seleccionado al menos alguna opción de las listas desplegables. Cuando esto sucede se genera una alerta en pantalla que dice: “Ingrese todos los parámetros requeridos”.
